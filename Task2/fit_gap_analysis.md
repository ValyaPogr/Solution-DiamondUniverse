# Fit/Gap-анализ для онлайн-аукциона «Diamond Universe»

## Список систем

| Система | Тип | Функциональность | Комментарий (что нужно доработать) |
| --- | --- | --- | --- |
| ИС «Алмазы» | Существующая | Отбор алмазов для цифрового аукциона, планирование работ, присвоение идентификаторов, подготовка ведомостей | Требуется реализовать признак «для цифрового аукциона», добавить экспорт данных в утверждённые форматы (в т. ч. Excel‑шаблоны), обеспечить формирование ведомостей под публикацию данных для клиентов |
| ИС «Аукцион» | Существующая | Автоматизация проведения аукциона (офлайн) | Необходимо добавить функционал публикации данных об алмазах для внешних клиентов, поддержку онлайн‑торгов, возможность отображения характеристик из оцифровки (размер, чистота, флуоресценция, фото/сканы и др.) |
| ИС «Логистика» | Существующая | Перевозка алмазов по стране и за границу | Нужно добавить сценарии перевозок «в лабораторию для оцифровки» и «возврат в офис после оцифровки», сформировать отдельные типы заданий и статусы для этих этапов, обеспечить передачу статусов для актирования в ИС «Бухгалтерия» |
| ИС «Бухгалтерия» | Существующая | Ведение контрактов, актирование и оплата услуг | Требуется доработать учёт новых типов услуг (оцифровка алмазов, специальные перевозки в лабораторию и обратно), добавить шаблоны актов и маршрутов согласования, обеспечить приём данных из ИС «Лаборатория» и ИС «Логистика» для автоматического формирования оплат |
| ИС «Лаборатория» (внешняя) | Новая | Оцифровка алмазов, формирование описания камней | Необходимо согласовать единый формат данных для передачи в ИС «Аукцион» и ИС «Бухгалтерия», реализовать защищённый канал передачи, определить SLA и правила контроля целостности данных |
| Облачные сервисы | Новые | Публикация данных для клиентов | Требуется заключить контракт с провайдером, настроить защищённый доступ, определить жизненный цикл данных, правила обновления и удаления, зафиксировать требования к безопасности и доступности |

## Список интеграций

| Интеграция | Тип | Реализация | Данные для передачи | Комментарий (что нужно доработать) |
| --- | --- | --- | --- | --- |
| ИС «Алмазы» → ИС «Логистика» | Существующая (расширяемая) | Автоматически (или полуавтоматически через выгрузку) | Список алмазов под перевозку, адреса, сроки, тип перевозки, признак «в лабораторию/возврат после оцифровки» | Нужно расширить сценарии перевозок, добавить новые типы заданий и статусов, реализовать обработку исключений и уведомлений при отклонениях |
| ИС «Лаборатория» → ИС «Аукцион» | Новая | Автоматически либо полуавтоматически (через выгрузку/загрузку по шаблону) | Описание алмазов: размер, чистота, флуоресценция в ультрафиолете, внутренние напряжения, трещины, форма и распределение включений, фото/сканы | Необходимо определить формат передачи (JSON/XML/Excel), согласовать схему данных, обеспечить контроль целостности и конфиденциальности, предусмотреть валидацию данных при загрузке |
| ИС «Лаборатория» → ИС «Бухгалтерия» | Новая | Автоматически либо вручную через утверждённый шаблон | Акт выполненных работ, объём услуг, стоимость, реквизиты, номер партии алмазов | Нужно согласовать процесс актирования, определить формат акта, маршрут согласования и сроки передачи, при ручной реализации — утвердить Excel‑шаблон и регламент заполнения |
| ИС «Логистика» → ИС «Бухгалтерия» | Существующая (расширяемая) | Автоматически | Задания на перевозку, статусы выполнения, акты оказанных услуг, тарифы, тип услуги | Требуется добавить новые типы услуг (доставка в лабораторию, возврат после оцифровки), обеспечить передачу дополнительных атрибутов для корректного актирования и расчёта стоимости |
